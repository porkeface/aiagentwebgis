## 1 引言

### 1.1 编写目的

本报告对系统进行全面需求分析，明确功能需求、非功能需求、接口需求及运行环境约束，为系统设计、编码实现及测试验收提供依据。

### 1.2 项目背景

随着自由行成为主流出行方式，用户对个性化、智能化旅行规划的需求日益迫切。传统方式存在信息过载、空间分析缺失、交互门槛高等痛点。本项目将AI Agent与WebGIS深度融合，用户以自然语言描述需求，系统自动完成空间分析、智能推荐与路线规划。

### 1.3 定义与缩略语

| 术语 | 说明 |
|------|------|
| AI Agent | 具备自主推理与工具调用能力的智能代理 |
| LangGraph | 基于图结构的LLM Agent编排框架 |
| POI | Point of Interest，兴趣点 |
| SSE | Server-Sent Events，服务器推送事件 |
| PostGIS | PostgreSQL空间数据库扩展 |
| ReAct | Reasoning + Acting推理-行动框架 |
| DBSCAN | 基于密度的空间聚类算法 |

## 2 任务概述

### 2.1 系统目标

**G1 — 自然语言意图解析**：准确解析用户输入，提取目的地、天数、偏好标签。

**G2 — 空间POI检索**：基于高德API + PostGIS执行POI搜索和空间查询。

**G3 — 多日行程规划**：通过十步确定性流水线生成合理多日行程。

**G4 — 地图可视化**：高德地图标注景点、绘制路线，与对话联动。

**G5 — 多轮对话**：基于LangGraph Checkpoint维护对话状态，支持上下文修改。

**G6 — 管理后台**：提供LLM配置、高德Key配置、数据管理的Web界面。

### 2.2 用户特点

| 用户类别 | 特征 | 核心需求 |
|---------|------|---------|
| 自由行旅客 | 18—45岁，追求个性化体验 | 个性化路线规划、避开拥挤 |
| 家庭出游者 | 携带老人/儿童 | 舒适路线、合理安排休息 |
| 背包客 | 预算敏感型 | 经济型推荐、距离最优 |
| 管理员 | 系统维护者 | 配置管理、数据监控 |

## 3 功能需求

### 3.1 意图分类模块（FR-001）

**描述**：接收用户自然语言输入，调用LLM进行分类，判断为行程规划/POI查询/一般对话。

**意图类型**：
- `trip_planning`：多日行程规划 → 进入规划流水线
- `poi_query`：POI信息查询 → 进入Agent ReAct循环
- `route_query`：两点间路线查询 → Agent ReAct循环
- `chat`：闲聊/问候 → Agent ReAct循环

**优先级**：P0（核心路径入口）

### 3.2 规划流水线模块（FR-002）

**描述**：针对`trip_planning`意图，执行确定性十步流水线。

**流水线步骤**：

1. **并行搜索**：5类关键词并行调用高德API搜索POI
2. **语义去重**：移除名称相同的重复POI
3. **多因子评分**：基于偏好匹配度、空间距离、评分、热度加权打分
4. **时长估算**：根据POI类别动态估算游览时长（60—180分钟）
5. **地理分区**：DBSCAN聚类分天 + 链式排序确保跨日连续
6. **障碍检测**：高德驾车距离vs直线距离对比，检测湖/山阻隔并拆分
7. **每日路线**：并行调用高德路径API规划每日最优游览顺序
8. **跨日交换**：Haversine检测边界POI跨日重分配
9. **验证提交**：数据结构校验 + 持久化到数据库
10. **LLM摘要**：生成自然语言行程介绍（≤300字）

**预选POI快速通道**：如果用户在消息中携带已选POI（地图选点），跳过搜索步骤，直接从分天开始执行。

**输入**：intent_city（城市）、intent_days（天数）、intent_preferences（偏好列表）

**输出**：daily_plans（每日行程，含POI列表、路线折线、分段信息）

**优先级**：P0（核心功能）

### 3.3 Agent工具模块（FR-003）

系统注册7个LangChain结构化工具：

| 工具名称 | 功能 | 调用API |
|---------|------|---------|
| search_pois | 按城市+类别搜索POI | 高德POI搜索API |
| search_nearby | 按坐标搜索周边POI | 高德周边搜索API |
| geocode | 地名↔坐标转换 | 高德地理编码API |
| plan_route | 两点/多点路线规划 | 高德路径规划API |
| score_pois | POI多因子评分 | 本地计算 |
| geo_partition | 地理分区分天 | 本地DBSCAN聚类 |
| submit_plan | 验证并提交行程 | 数据校验+持久化 |

**并行策略**：search_pois、search_nearby、geocode、score_pois标记为可并行调用。

**超时控制**：每个工具调用有30秒超时保护。

**优先级**：P0

### 3.4 地图可视化模块（FR-004）

**描述**：基于高德JS API 2.0实现地图渲染和交互。

**功能点**：
- POI Marker标注（按类别着色、区分图标）
- 27px圆形Marker带白色边框
- 路线折线绘制（按天着色）
- 点击Marker查看POI详情
- 地图与对话联动（收到新POI自动更新）

**优先级**：P0

### 3.5 对话交互模块（FR-005）

**描述**：用户与AI Agent的对话交互界面。

**功能点**：
- 消息气泡（用户/AI/系统三种样式）
- SSE流式接收（打字机效果）
- 思考过程展示
- 工具调用状态指示
- PlanCard行程卡片渲染
- 地图选点规划入口
- 会话历史管理

**SSE事件类型**（20种）：thinking、tool_calling、poi_result、route_result、plan_summary、text、error、progress、done、intent_detected、searching、candidates_ready、scoring、clustering、day_routing、critic_review、critic_result、routing、day_routed、validating

**优先级**：P0

### 3.6 用户认证模块（FR-006）

**描述**：JWT无状态认证。

**功能点**：
- 用户注册（用户名+密码）
- 用户登录（返回JWT Token）
- Token验证中间件
- 管理员权限控制

**优先级**：P0

### 3.7 管理后台模块（FR-007）

**描述**：管理员配置和数据管理界面。

**功能点**：
- **模型配置**：LLM供应商切换、模型选择、API Key配置（支持热重载）
- **高德地图配置**：后端AMAP_API_KEY + 前端VITE_AMAP_KEY独立管理
- **用户管理**：列表、删除
- **数据管理**：行程/POI/会话统计与浏览

**热重载机制**：
- LLM配置变更 → 清除编译图缓存，下次请求自动重建
- AMAP_API_KEY变更 → 清除AmapService单例
- VITE_AMAP_KEY变更 → 需重启前端Vite服务

**优先级**：P1

### 3.8 行程管理模块（FR-008）

**描述**：用户行程的查看与管理。

**功能点**：
- 行程列表（按状态筛选）
- 行程详情（按天展示POI + 路线）
- 行程删除

**优先级**：P0

## 4 非功能需求

### 4.1 性能需求

| 指标 | 要求 |
|------|------|
| 意图分类响应 | ≤3秒 |
| POI检索响应 | ≤2秒 |
| 规划流水线完成 | ≤30秒（含高德API调用） |
| 地图渲染 | 流畅（50个Marker以内） |
| 首屏加载 | ≤3秒 |

### 4.2 安全性需求

- JWT Token认证保护API端点
- API Key通过环境变量管理，管理员界面密码输入框+掩码显示
- 速率限制：Agent接口20次/分钟，认证接口10次/分钟
- 配置白名单：仅允许更新安全可热重载的配置项

### 4.3 可靠性需求

- LLM API失败时自动重试1次
- 工具调用超时30秒自动取消
- 规划流水线任意步骤异常不影响整体（记录日志+返回部分结果）
- 数据库操作使用SQLAlchemy事务管理

### 4.4 可扩展性需求

- LLM适配：通过ChatOpenAI统一接口支持DashScope/DeepSeek
- 工具插件化：新增工具只需实现StructuredTool接口
- 配置白名单：新增可热重载配置只需加入_ALLOWED_UPDATE_KEYS

### 5.2 SSE通信协议

请求 `POST /api/v1/agent/chat`：
```json
{"message": "帮我规划杭州三日游", "session_id": "uuid"}
```

响应流（text/event-stream）：
```
event: message
data: {"type":"intent_detected","data":{"intent":"trip_planning",...}}

event: message  
data: {"type":"searching","data":{"message":"正在搜索杭州的景点..."}}

event: message
data: {"type":"candidates_ready","data":{"pois":[...],"count":42}}

event: message
data: {"type":"clustering","data":{"message":"正在按地理分区规划3天行程..."}}

event: message
data: {"type":"day_routing","data":{"message":"正在规划每日路线..."}}

event: message
data: {"type":"route_result","data":{"daily_plans":[...]}}

event: message
data: {"type":"plan_summary","data":{...}}

event: message
data: {"type":"text","data":"已为您生成杭州三日游方案"}

event: done
data: {}
```

### 5.3 外部API接口

| 接口 | 用途 | 调用频率 |
|------|------|---------|
| 高德POI搜索 | 搜索景点、周边查询 | 规划时调用 |
| 高德地理编码 | 地名→坐标转换 | 城市定位时调用 |
| 高德路径规划 | 驾车/步行/公交路线 | 规划时每段一次 |
| DashScope/DeepSeek | LLM推理 | 每次对话调用 |
| OSM瓦片服务 | 地图底图（备用） | 不依赖高德时可切换 |
