## 1. 页面结构规划

### 1.1 整体页面架构

系统采用 **SPA（单页应用）** 架构，共分为以下核心页面/视图：

```
┌──────────────────────────────────────────────────────┐
│                     顶部导航栏                         │
│  Logo  |  首页  |  地图探索  |  我的行程  |  个人中心    │
├──────────────────────────────────────────────────────┤
│                                                      │
│                    路由视图区域                         │
│                                                      │
│  /home        → 首页（系统介绍 + 快速入口）              │
│  /explore     → 地图探索页（核心页面）                    │
│  /chat        → AI 对话规划页（核心页面）                 │
│  /trip/:id    → 行程详情页                               │
│  /profile     → 个人中心（偏好管理）                      │
│  /about       → 关于系统                                 │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### 1.2 各页面核心功能

#### ① 首页 `/home`
| 模块 | 说明 |
|------|------|
| Hero 区 | 全屏地图背景 + 搜索框 + 一句话 Slogan，支持直接在搜索框输入目的地 |
| 快速入口 | 3~4 个卡片：「智能推荐」「自定义探索」「热门路线」「我的行程」|
| 数据展示 | 动画计数器：收录景点数 / 规划路线数 / 服务用户数 |
| 技术展示 | 简短动画演示系统核心功能（地图 → 对话 → 路线生成）|

#### ② 地图探索页 `/explore`（核心页面，占比最大）
| 模块 | 说明 |
|------|------|
| **左侧边栏**（320px）| 景点搜索 + 筛选面板 + 景点列表 + 图层控制 |
| **中央地图**（自适应）| 全功能交互地图，承载所有空间可视化 |
| **右侧信息面板**（360px，可折叠）| 景点详情 / POI 卡片 / 路线概览 |
| **底部时间轴**（可选）| 时空约束可视化：展示景点开放时段 |

#### ③ AI 对话规划页 `/chat`（核心页面）
| 模块 | 说明 |
|------|------|
| **左侧对话区**（480px）| ChatGPT 风格对话界面 |
| **右侧地图区**（自适应）| 实时联动地图，Agent 推荐结果即时可视化 |
| **底部方案卡片栏**（可折叠）| 当前推荐方案的卡片化展示 |

#### ④ 行程详情页 `/trip/:id`
| 模块 | 说明 |
|------|------|
| 顶部行程概览 | 总距离、总时间、景点数量、难度评分 |
| 地图路线展示 | 完整路线 + 各景点标注 + 时间轴 |
| 时间线列表 | 按时间顺序的景点列表，含交通方式与耗时 |
| 操作按钮 | 导出 PDF / 分享 / 修改 / 导航 |

#### ⑤ 个人中心 `/profile`
| 模块 | 说明 |
|------|------|
| 偏好设置 | 旅行风格（文化/自然/美食/冒险）、预算、出行方式 |
| 历史行程 | 历史规划列表 + 收藏景点 |
| 足迹地图 | 小地图展示去过的景点（打卡效果）|

## 2. 地图交互设计

### 2.1 图层体系

系统地图采用 **多图层叠加** 架构，图层之间可独立开关：

```
┌─────────────────────────────────────┐
│  Layer 7: 用户标注图层（手绘路线）      │  ← 用户可绘制
├─────────────────────────────────────┤
│  Layer 6: 时空约束图层（时段色带）      │  ← 景点开放时段
├─────────────────────────────────────┤
│  Layer 5: 推荐路线图层（动态动画）      │  ← Agent 推荐
├─────────────────────────────────────┤
│  Layer 4: 交通网络图层                │  ← 道路/公交/地铁
├─────────────────────────────────────┤
│  Layer 3: 热力图/聚类图层             │  ← 景点密度分布
├─────────────────────────────────────┤
│  Layer 2: 区域边界图层                │  ← 行政区划/景区边界
├─────────────────────────────────────┤
│  Layer 1: 基础底图                    │  ← 卫星图/矢量图/地形图
└─────────────────────────────────────┘
```

### 2.2 各图层详细说明

| 图层 | 内容 | 技术实现 | 交互 |
|------|------|----------|------|
| **基础底图** | 卫星影像 / 矢量街道 / 地形 | 地图 SDK 内置瓦片 | 切换底图样式 |
| **区域边界** | 行政区划、景区边界 | GeoJSON Polygon | 点击高亮、悬浮提示 |
| **热力图/聚类** | 景点空间密度分布 | WebGL Heatmap / Supercluster | 动态半径调节、开关 |
| **交通网络** | 道路、公交、地铁线路 | 矢量线图层 | 路径高亮 |
| **推荐路线** | Agent 生成的旅行路线 | Polyline + 动画 | 路线对比、分步导航 |
| **时空约束** | 景点开放时间的可视化 | 色带/环形进度条覆盖物 | 时间滑块联动 |
| **用户标注** | 手绘标记、自定义路线 | Drawing Tools | 绘制、编辑、删除 |

### 2.3 用户交互方式

| 交互方式 | 触发操作 | 效果 |
|----------|----------|------|
| **缩放** | 滚轮/双指缩放 | 自动切换图层显示级别（<12 显示聚类，>14 显示单个 POI）|
| **点击 POI** | 单击景点标记 | 弹出信息卡片 + 右侧面板联动 |
| **悬浮预览** | 鼠标悬浮 | 迷你 Tooltip（景点名 + 评分 + 缩略图）|
| **框选区域** | 矩形/多边形框选 | 统计区域内景点数 → "AI 推荐此区域路线" |
| **画圆搜索** | 圆形工具 | "以我为中心，3km 内有什么好玩的？" |
| **路线绘制** | 手动连线 | 自定义路线，叠加到地图 |
| **时间滑块** | 拖动时间轴 | 地图上的时空约束图层实时更新 |
| **图层切换** | 控制面板勾选 | 按需显示/隐藏各图层 |
| **双屏对比** | 分屏模式 | 左右地图分别展示不同推荐方案 |
| **3D 俯仰** | Ctrl + 拖动 | 切换 2D/3D 视角（Mapbox GL 支持）|

### 2.4 Agent 推荐结果的地图可视化

```
Agent 推荐流程：

用户对话输入 → Agent 分析 → 返回推荐 JSON
                                  │
                    ┌─────────────┼─────────────┐
                    │             │             │
                    ▼             ▼             ▼
              景点标记点      推荐路线        热力覆盖
           (Marker + 弹跳动画)  (Polyline)   (Heatmap)
                    │             │             │
                    └─────────────┼─────────────┘
                                  │
                                  ▼
                            地图飞行动画
                        (flyTo 聚焦推荐区域)
```

- **景点推荐**：新景点以 **弹跳动画** 依次出现在地图上，间隔 200ms，形成"逐个点亮"效果
- **路线推荐**：路线以 **虚线流动动画** 从起点向终点逐步绘制，配合粒子流动效果
- **区域推荐**：推荐区域以 **渐变色填充 + 呼吸效果** 高亮显示
- **多方案对比**：不同方案用不同颜色路线同时显示，底部卡片可切换

## 3. 空间可视化亮点（答辩加分项）

### 3.1 景点空间聚类可视化 🔥

**效果描述**：
- 低缩放级别下，景点以 **聚合气泡** 显示（数字标注内含景点数）
- 气泡大小反映聚类规模，颜色从绿（少）→ 黄 → 红（多）渐变
- 放大时聚合气泡 **爆炸散开** 为单个景点标记（过渡动画 300ms）
- 同时支持 **热力图模式** 切换：用 WebGL 渲染的连续密度面

**技术实现**：
```
supercluster 库 → 前端聚合计算
mapbox-gl heatmap layer → WebGL 热力渲染
自定义 Marker 组件 → 聚类气泡
```

**答辩亮点**：展示空间聚类分析能力，体现 WebGIS 空间分析的核心价值

### 3.2 推荐路线的动态绘制动画 🔥

**效果描述**：
- 路线从起点到终点 **逐步绘制**，像"铅笔画线"效果
- 沿线有 **流动粒子**（光点）表示行进方向
- 每个景点节点 **脉冲发光** 一次（到达提示）
- 配合 **进度条** 显示游览时间进度

**技术实现**：
```javascript
// 路线逐步绘制
map.addLayer({
  id: 'route-animate',
  type: 'line',
  source: routeSource,
  paint: {
    'line-dasharray': [0, 4, 3],
    'line-color': '#4F46E5',
    'line-width': 4
  }
});

// 使用 turf.js 沿线插值 + requestAnimationFrame
// 实现粒子流动效果
```

**答辩亮点**：动态路线是最直观的 WebGIS 展示效果

### 3.3 景点 POI 信息卡片 🔥

**效果描述**：
- 点击景点后，地图上弹出 **毛玻璃风格浮层卡片**
- 卡片包含：景点封面图、名称、评分（星级）、简介（2行）
- 底部操作：「查看详情」「添加到行程」「AI 分析」
- 右侧面板同步展开 **完整信息**：
  - 开放时间（带时间轴可视化）
  - 门票价格
  - 实时客流（热力指示器）
  - 周边设施（餐饮/停车场/卫生间图标）
  - 用户评价 Top3

**卡片视觉设计**：
```
┌──────────────────────────────┐
│ ┌──────────────────────────┐ │
│ │                          │ │ ← 封面图（圆角）
│ │      景点封面图            │ │
│ │                          │ │
│ └──────────────────────────┘ │
│ ★ 4.8    文化古迹    ¥60     │ ← 评分 + 标签 + 价格
│                              │
│ 颐和园是中国现存最大的       │ ← 简介
│ 皇家园林之一...              │
│                              │
│ 🕐 06:30-18:00  📍 距你2.3km │ ← 开放时段 + 距离
│                              │
│ [  查看详情  ] [ AI分析 ]     │ ← 操作按钮
└──────────────────────────────┘
```

### 3.4 时空约束可视化 🔥

**效果描述**：
- 地图上每个景点标记外围显示 **环形时间进度条**
  - 绿色弧 = 当前处于开放时间
  - 灰色弧 = 尚未开放
  - 红色弧 = 已关闭
- 底部 **时间轴滑块**：拖动时地图上的色环实时更新
- 路线上叠加 **时间色带**：每个路段用颜色表示建议游览时段
  - 蓝色 = 上午推荐
  - 橙色 = 下午推荐
  - 紫色 = 傍晚/夜间推荐
- AI 规划时自动考虑时间约束，地图上可视化展示 **时间合理性**

**答辩亮点**：体现时空一体化分析能力，是 WebGIS 区别于普通地图应用的关键

### 3.5 用户偏好与推荐结果对比 🔥

**效果描述**：
- 雷达图（Spider Chart）展示用户偏好维度：文化/自然/美食/冒险/休闲/预算
- 推荐结果同样用雷达图展示
- 两张雷达图 **叠加对比**，透明度 50%
- 地图上用 **颜色编码** 显示匹配度：
  - 绿色标记 = 高度匹配（>80%）
  - 黄色标记 = 中度匹配（50%-80%）
  - 灰色标记 = 低匹配（<50%）
- 支持 "探索模式"：临时放宽某维度约束，地图即时刷新

### 3.6 额外加分可视化效果

| 效果 | 说明 | 技术 |
|------|------|------|
| **3D 建筑白模** | 景点周边建筑立体展示 | Mapbox GL `fill-extrusion` |
| **轨迹回放** | 用户历史足迹动画回放 | Turf.js + 时间插值 |
| **等时圈** | "15分钟能到达哪些景点" | 等时线计算 + Polygon 渲染 |
| **OD 迁徙图** | 景点间客流方向的弧线图 | Arc 曲线 + 流动动画 |
| **天气叠加** | 天气信息叠加到地图上 | 天气 API + 粒子效果（雨/雪）|

### 4.3 对话与地图联动机制

```javascript
// 消息-地图联动事件总线
const MapChatBridge = {
  // Agent 推荐景点时
  onPoiRecommend(pois: Poi[]) {
    map.flyTo({ center: bounds.center, zoom: 12 });
    pois.forEach((poi, i) => {
      setTimeout(() => addBounceMarker(poi), i * 200);
    });
  },

  // Agent 推荐路线时
  onRouteRecommend(route: Route) {
    animateRouteDrawing(route);        // 路线绘制动画
    highlightPoisOnRoute(route.pois);  // 高亮沿线景点
    showTimelinePanel(route);          // 显示时间轴面板
  },

  // 用户点击地图上的景点时
  onMapPoiClick(poi: Poi) {
    sendMessageToChat(`我对 ${poi.name} 感兴趣`);
    scrollToBottom();
  },

  // 用户在地图框选区域时
  onMapSelection(bbox: BBox) {
    sendMessageToChat(
      `请推荐这个区域内的景点：${formatBBox(bbox)}`
    );
  },

  // 用户点击对话中的方案时
  onSchemeClick(schemeId: string) {
    showSchemeOnMap(schemeId);
  }
};
```

## 5. 技术方案

### 5.1 技术栈总览

```
┌─────────────────────────────────────────────────────┐
│                    前端技术栈                         │
├─────────────────────────────────────────────────────┤
│                                                     │
│  框架层     Vue 3 + TypeScript + Vite               │
│  UI 组件    Element Plus (表单/布局) + 自定义组件     │
│  地图引擎   Mapbox GL JS (主地图)                    │
│  地图封装   vue-mapbox-gl (Vue3 组件封装)             │
│  空间分析   Turf.js (空间计算) + Supercluster (聚类)  │
│  图表       ECharts 5 (雷达图/柱状图/时间轴)          │
│  状态管理   Pinia (轻量级、TypeScript 友好)           │
│  路由       Vue Router 4                             │
│  HTTP       Axios + EventSource (SSE)                │
│  样式       UnoCSS / TailwindCSS + CSS Variables     │
```

### 5.2 地图 SDK 选型对比

| 维度 | Mapbox GL JS | 高德 JS API | 百度 GL JS | Leaflet |
|------|-------------|------------|-----------|---------|
| **渲染** | WebGL | WebGL/Canvas | WebGL | Canvas |
| **3D 支持** | ⭐⭐⭐ 原生 3D | ⭐⭐ 部分支持 | ⭐⭐ 部分支持 | ⭐ 需插件 |

| **路线动画** | ⭐⭐⭐ 原生支持 | ⭐⭐ 需手动 | ⭐⭐ 需手动 | ⭐ 插件 |
| **中国数据** | ⭐⭐ 需注意合规 | ⭐⭐⭐ 原生 | ⭐⭐⭐ 原生 | ⭐⭐ 底图源 |
| **免费额度** | 50K 加载/月 | 个人免费 | 个人免费 | 完全免费 |
| **学习曲线** | 中等 | 低 | 低 | 低 |
| **综合评分** | **⭐⭐⭐⭐⭐** | ⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ |

**最终选择：Mapbox GL JS**

理由：
1. **WebGL 渲染性能最佳**：热力图、3D 建筑、路线动画均原生支持
2. **自定义能力最强**：Mapbox Studio 可视化配置样式，表达力最强
3. **答辩效果最好**：3D 视角、飞行动画、流畅过渡最能打动老师
4. **生态最丰富**：Turf.js 无缝集成、丰富的 GL 插件
5. **学术认可度高**：WebGIS 课程答辩中 Mapbox 比国产地图更有"技术感"

### 5.4 状态管理方案

```
Pinia Store 架构：

├── useMapStore        地图状态（中心、缩放、图层可见性、当前选中要素）
├── usePoiStore        景点数据（POI 列表、筛选条件、聚类状态）
├── useRouteStore      路线数据（推荐路线、自定义路线、当前方案）
├── useChatStore       对话状态（消息列表、对话上下文、流式响应）
├── useTripStore       行程数据（行程列表、行程安排、偏好设置）
├── useLayerStore      图层控制（图层开关、透明度、样式）
└── useTimeStore       时间状态（当前时间、时间范围、时段筛选）
```

Store 间通过 **组合式函数（Composables）** 协作：

```typescript
// composables/useMapPoiBridge.ts
// 桥接地图与 POI 数据
export function useMapPoiBridge() {
  const mapStore = useMapStore();
  const poiStore = usePoiStore();

  // 监听 POI 变化 → 更新地图标记
  watch(() => poiStore.visiblePois, (newPois) => {
    mapStore.updateMarkers(newPois);
  });

  // 监听地图视口 → 更新 POI 加载范围
  watch(() => mapStore.viewport, (viewport) => {
    poiStore.loadPoisInBounds(viewport.bounds);
  });
}
```

### 5.5 与后端 Agent 通信方案

**主方案：SSE（Server-Sent Events）+ REST API 混合**

```
┌─────────────────────────────────────────────────┐
│                 通信架构                          │
├─────────────────────────────────────────────────┤
│                                                 │
│  对话消息 ──→ SSE (流式)                         │
│               单向流：后端 → 前端                 │
│               场景：Agent 思考过程 + 结果逐步输出   │
│               优势：简单、兼容性好、断线自动重连     │
│                                                 │
│  景点/路线 ──→ REST API                          │
│               请求-响应模式                        │
│               场景：POI CRUD、路线保存、历史查询    │
│                                                 │
│  实时协作 ──→ WebSocket (可选)                    │
│               双向通信                            │
│               场景：多人协作规划（扩展功能）         │
│                                                 │
└─────────────────────────────────────────────────┘
```

**SSE 通信流程**：

```
前端                              后端 Agent
 │                                   │
 │  POST /api/chat/stream            │
 │  { message: "推荐北京三日游" }      │
 │ ─────────────────────────────────→ │
 │                                   │  Agent 思考...
 │  event: thinking                  │
 │  data: "正在分析您的需求..."        │
 │ ←───────────────────────────────── │
 │                                   │
 │  event: poi_recommend             │  ← 前端收到后触发地图标注
 │  data: { pois: [...] }           │
 │ ←───────────────────────────────── │
 │                                   │
 │  event: route_recommend           │  ← 前端收到后触发路线动画
 │  data: { route: {...} }          │
 │ ←───────────────────────────────── │
 │                                   │
 │  event: done                      │
 │  data: [DONE]                     │
 │ ←───────────────────────────────── │
```

**为什么不用 WebSocket**：
- 此项目以 **Agent 单向输出** 为主（用户发消息 → Agent 回复），SSE 足够
- SSE 基于 HTTP，无需额外协议升级，部署更简单
- SSE 浏览器原生支持 `EventSource` API，代码更简洁
- 断线自动重连机制内建
